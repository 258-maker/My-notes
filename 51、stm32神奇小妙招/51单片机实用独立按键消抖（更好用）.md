# bit与sbit
>在 Keil C51（针对 51 单片机的 C 语言编译器）中，`bit`是专门用来定义**位变量**的关键字 —— 简单说，`bit`变量只占用**1 个二进制位**（0 或 1），是 51 单片机特有的、极致节省内存的变量类型。

**形象比喻**：
- `unsigned char`像一个能装 8 杯水的杯子，而`bit`只像一个能装 1 滴水的小勺子 —— 如果只需要记录 “按键按下（0）/ 松开（1）”“LED 亮（0）/ 灭（1）” 这种二选一的状态，用`bit`比用`char`节省 7 个位的内存（51 单片机总 RAM 只有几百字节，省内存很重要）
## `bit`的使用注意事项
1. **仅支持 Keil C51 编译器**：`bit`是 51 单片机专属的关键字，在标准 C（如 STM32 的 GCC 编译器）中不存在，标准 C 中用`_Bool`或`bool`（需包含`stdbool.h`）代替;
2. **`bit`变量的赋值 / 判断只能是 0 或 1**;
3. **`bit`变量不能用于数组 / 指针**：51 单片机的位操作有硬件限制，不能定义`bit arr[5];`（位数组）或`bit *p;`（位指针），只能单个定义。
## 和`sbit`的区别

你代码中还有`sbit key4=P2_7;`，注意`sbit`≠`bit`：

- `bit`：定义**普通位变量**（存在 RAM 中，可读写）；
- `sbit`：定义**单片机引脚的位别名**（绑定硬件引脚，如 P2_7），用于操作硬件。
```c
sbit key4=P3^0;//一定是P3^0而不是P3_0
```
# 消抖程序
```c
keybuff     //存储按键连续8次扫描状态缓冲区，定义为unsigned char类型
keySta      //通过keybuff8次扫描后确定的按键状态
backup      //用于保存上一次按键状态，备份KeySta，用于和KeySta比较判断当前时按下还是弹起，定义为bit类型
```
       0xFF         <<1        0xFE
    1111 1111      <<1     1111 1110
```c
#include <REGX52.H>
sbit key4=P3^0;
void Timer0_Init(void);

bit KeySta=1;
bit backup=1;
void main()
{
	Timer0_Init();
	P2=0xFF;
	while(1)
	{
		if(KeySta!=backup)
		{
			if(backup==0)
			{
				P2=~P2;
			}
			backup=KeySta;
		}
	
	
	}



}

void Timer0_Init(void)		//2毫秒@11.0592MHz
{
	TMOD &= 0xF0;			//设置定时器模式
	TMOD |= 0x01;			//设置定时器模式
	TL0 = 0xCD;				//设置定时初始值
	TH0 = 0xF8;				//设置定时初始值
	TR0 = 1;				//定时器0开始计时
	EA=1;
	ET0=1;
}

void My_Tim0() interrupt 1
{
	static unsigned char keybuff=0xFF;
	TL0 = 0xCD;
	TH0 = 0xF8;	

	keybuff=(keybuff << 1)|key4;
	if(keybuff==0x00)
	{
		KeySta=0;
	}
	else if(keybuff==0xFF)
	{
		KeySta=1;
	}
	else
	{
	}
}
```
