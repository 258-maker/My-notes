# 通信的基本概念
## 串行通信与并行通信
1. 串行通信
![](图片/IMG_20260214_204754.jpg)
- 特点：传输线很少
2. 并行通信
![](图片/IMG_20260214_205109.jpg)
- 特点控制简单，传输速度快

## 异步通信与同步通信
1. 异步通信
![](图片/IMG_20260214_205447.jpg)
- 要求发送设备与接收设备的时钟尽可能保持一致，但不用严格一致

2. 同步通信
![](图片/IMG_20260214_205951.jpg)
- 外同步：通过发送方的时钟来控制接收方的时钟；
- 自同步：通过发送方发送数据和时钟来控制接收方；

## 单工、半双工与全双工通信
1. 单工通信
![](图片/IMG_20260214_210703.jpg)
- 只能一个方向发送
2. 半双工通信
![](图片/IMG_20260214_210718.jpg)
- 可以按照两个方向进行传输，但不能同时进行
3. 全双工通信
![](图片/IMG_20260214_211254.jpg)
- 可以进行两个方向同时传输

## 通信速率（比特率）
- 比特率单位：位/秒（bps）
- 假设一秒发送240个字符，假设一个字符由一个起始位、八个数据位、一个停止位组成，那么比特率为（240X10）/1，即2400bps
## 波特率
- 每秒钟传输了多少个码源
- 基本可以把波特率看成比特率

# 串口通信简介
## 接口标准
- 接口：TXD、RXD、GND；
- 接法：TXD——RXD、RXD——TXD、GND——GND；
- ==通信协议：通常遵循96-N-8-1格式即（9600bps，N——无校验位（奇校验、偶校验、无校验），8——8位数据位，1——停止位长度）==

## 串口内部结构
![](图片/IMG_20260214_214457.jpg)

- TXD（P3.1），RXD（P3.0）
- SBUF：数据缓存区
- 波特率产生是由TH1和TL1（定时器1的功能）产生，配置方式2
- SMOD标志位控制是否使用倍频，置1倍频。

# 串口相关寄存器
## 串口控制寄存器SCON
![](图片/IMG_20260214_215544.jpg)
- SM0、SM1：用来确定串口通信的工作方式，通常使用方式1
- SM2：多机通信控制位，主要是方式2，方式3，为1时来控制RB8为0、为1以此来达到多机通信的目的
- RB8：为1是表示激活RI，为0时表示不激活RI
- TB8：为1是表示激活TI，为0时表示不激活TI
- REN：允许接收位
- TI：发送中断请求标志位，在方式0，当串行发送数据第8位结束时，由内部硬件自动置位，即T1=1时，向主机请求中断，响应中断后必须用软件复位，即TI=0。在其他方式中，则在停止位开始发送时由内部硬件置位，必须用软件复位。初始化配置给0。
- RI：接收中断请求标志位，在方式0，当串行接收到第8位结束时，由内部硬件自动置位，即RI=1，向主机请求中断，响应中断后必须用软件复位，即RI=0。在其他方式中，串行接收到停止位的中间时刻由内部硬件置位，即RI=1（例外情况见SM2说明），必须由软件复位，即RI=0。初始化配置给0。
**设置为：01010011**
## 电源控制寄存器PCON
![](图片/IMG_20260214_221508.jpg)
- SMOD：对倍频的选择
# 计算串口波特率
方式0的波特率= fosc / 12
方式2的波特率=（2SMOD/64）X fosc
方式1的波特率=（2SMOD/32）X （T1溢出率）
方式3的波特率=（2SMOD/32）X （T1溢出率）
**其中T1溢出率=fosc / 12[256-(TH1)]** TH1为初值

# 串口配置步骤
1. 确定串口工作方式（SCON寄存器）；
2. 确定T1的工作方式（TMOD寄存器）；
3. 计算T1的初值（设定波特率），装载TH1、TL1；
4. 启动T1（TCON中的TR1位）；
5. 如果使用中断，需开启串口中断控制位（IE寄存器）

```c
void uart_init(u8 baud)
{
	SCON=0x50;//设置为工作方式1
	PCON=0x80;//波特率加倍
	TMOD|=0x20;//设置计数器工作方式2
	TH1=baud;//计数器初始值设置
	TL1=baud;
	ES=1;//打开接收中断
	EA=1;//打开总中断
	TR1=1；//打开计数器
}
```
